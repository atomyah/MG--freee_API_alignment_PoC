# MG → xxxxx人事労務 PoC

## 概要

MGの従業員・勤怠データを xxxxxx人事労務 API 経由で連携する
Proof of Concept（概念実証）です。

```
┌─────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  MG         │ ──→ │  変換レイヤー     │ ──→ │ xxxxxx人事労務    │
│ (ダミーJSON) │     │  (Python scripts) │     │ (テスト事業所)   │
└─────────────┘     └──────────────────┘     └─────────────────┘
```

---

## ファイル構成

```
mg-xxxxxx-poc/
├── README.md                    ← このファイル
├── config.py                    ← ★ 自分の認証情報を入力
├── mg_dummy_data.json           ← MGダミーデータ（従業員5名 + 勤怠1ヶ月）
├── step1_register_employees.py  ← Step 1: 従業員登録
├── step2_input_attendance.py    ← Step 2: 勤怠投入
├── step3_verify.py              ← Step 3: 投入結果の確認
└── employee_mapping.json        ← (自動生成) MG ID → freee ID の対応表
```
※ config.pyとmd_dummy_data.jsonはこのレポジトリにはありません（to 関係者様、別途送付します）

## ダミーデータの設計

### 従業員 5名（mg_dummy_data.json）

| MG ID  | 氏名     | 雇用形態 | 基本給/時給  | 特徴                   |
|--------|----------|----------|-------------|------------------------|
| MG-001 | 田中 太郎 | 月給     | ¥280,000    | 標準的な社員、残業あり   |
| MG-002 | 佐藤 花子 | 月給     | ¥300,000    | 有給取得2回             |
| MG-003 | 鈴木 一郎 | 時給     | ¥1,500/h    | 時短勤務(10-16時)、欠勤1回 |
| MG-004 | 高橋 美咲 | 月給     | ¥260,000    | 午前半休1回             |
| MG-005 | 伊藤 健太 | 月給     | ¥350,000    | 深夜残業あり(22時退勤)  |

### 勤怠データ（2025年2月）

各従業員 18〜20日分の日次勤怠。以下のパターンを網羅:

- **通常出勤** : 所定時間どおり
- **残業**     : 法定内(1h〜3h)、深夜帯(22:00超過)
- **有給休暇** : 全日休
- **半休**     : 午前半休（有給0.5日消化）
- **欠勤**     : 無給欠勤
- **祝日**     : 建国記念の日(2/11)、振替休日(2/24) → freee側自動判定

## セットアップ手順

### 前提条件

- Python 3.8以上
- xxxxxx 開発者アカウント + テスト事業所 + テストアプリ 作成済み
  （https://developer.freee.co.jp/startguide 参照）
- アクセストークン取得済み
- company_id（事業所ID）取得済み

### 1. 依存ライブラリのインストール

```bash
source .venv/bin/activate
pip install requests
```

### 2. config.py を編集

```python
CLIENT_ID     = "あなたのClient ID"
CLIENT_SECRET = "あなたのClient Secret"
ACCESS_TOKEN  = "あなたのアクセストークン"
COMPANY_ID    = 1234567  # 数値で入力
```

### 3. xxxxxx Web画面での事前設定（重要）

テスト事業所で以下が設定済みか確認:

1. **締め日支払い日**（設定 → 締め日支払い日）https://p.secure.freee.co.jp/settings#/
   - テスト事業所作成時にデフォルトで1つ存在するはず
   - 例: 末日締め / 翌月25日払い

2. **勤務・賃金設定**（設定 → 勤務・賃金）https://p.secure.freee.co.jp/settings#/
   - 所定労働時間: 8時間/日
   - 所定労働日数: 20日/月

## 実行手順

### Step 1: 従業員登録

```bash
python step1_register_employees.py
```

出力例:
```
  --- 田中 太郎 (MG-001) ---
  ✅ 登録成功! freee employee_id = 12345

  完了！ 5/5 名の登録に成功
```

→ `employee_mapping.json` が自動生成されます
→ ２回実行すると従業員データが重複して入ってしまいます。

### Step 2: 勤怠データ投入

```bash
python step2_input_attendance.py
```

出力例:
```
  --- 田中 太郎 (MG-001 → freee:12345) ---
  2025-02-03: ✅ (8h00m)
  2025-02-04: ✅ (9h30m) 残業1.5h
  ...
  完了！ 成功: 85件  スキップ: 5件  失敗: 0件
```

### Step 3: 投入結果の確認

```bash
python step3_verify.py
```

→ API経由で月次サマリを取得し、出勤日数・残業時間等を表示

### Step 4: xxxxxxx Web画面で給与計算

1. xxxxx人事労務にログイン
2. [給与明細] タブ → 2025年2月を選択
3. [給与計算を実行] をクリック
4. 各従業員の給与明細を確認

## トラブルシューティング

### アクセストークンが期限切れ

```
"expired_access_token" エラー
```
→ アプリ管理画面でトークンを再取得、config.py を更新

### 締め日支払い日エラー

```
"締め日支払い日を選択してください"
```
→ xxxxxx Web画面 → 設定 → 締め日支払い日 で設定を確認

### 権限エラー

```
"アクセスする権限がありません"
```
→ アプリ管理 → 権限設定 で以下を有効化:
  - 従業員情報の参照・更新
  - 勤怠情報の参照・更新
→ 権限変更後はトークン再取得が必要
